# Dandelifeon Optimizer

A high-performance, multi-threaded engine designed to find mathematically optimal patterns for the **Dandelifeon** flower from the *Botania* mod.

---

## How it Works
The optimizer explores the chaotic state-space of the Game of Life using a **Multi-threaded Hill Climbing** approach.

### Key Features
- **Surgical Decimation:** Automatically prunes redundant blocks once the mana cap is reached.
- **Genetic Cross-pollination:** Shares successful clusters between different thread "islands" to prevent stagnation.
- **6 Symmetry Modes:** Explores various topologies, from rotational to axial-mirroring.

---

## Dandelifeon Rules
This variant of Conway's Game of Life uses unique mechanics found in Botania:

1. **Grid:** 25x25 limited board.
2. **Absorption:** 3x3 center zone. Any cell entering triggers a `Wipe` and yields mana.
3. **Inheritance:** New cells take the age of the oldest neighbor + 1.
4. **Efficiency:** $Mana = Cells \times Age \times 150$.

> [!NOTE]  
> 8 cells in the center is the mathematical limit, but 6 cells is the practical ceiling for most stable patterns.

---

## Configuration
Tailor the search in `main.cpp` using the `startCustomOptimization` function:

| Parameter | Default | Description |
| :--- | :--- | :--- |
| `maxTicks` | 60 | Maximum generations before pattern expiration |
| `manaPerTick` | 150 | Age multiplier (Vanilla = 150) |
| `manaCap` | 50000 | Internal buffer limit of the flower |
